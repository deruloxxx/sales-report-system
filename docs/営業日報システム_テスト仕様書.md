## 1. テスト概要

### 1.1 目的

営業日報システムの全機能が要件定義・画面定義書・API仕様書どおりに動作することを確認する。

### 1.2 テスト種別

| 種別 | 対象 | 概要 |
|------|------|------|
| 機能テスト | 全画面 | 各機能が正しく動作すること |
| バリデーションテスト | 入力画面 | 入力チェックが正しく動作すること |
| 権限テスト | 全画面・全API | ロール別のアクセス制御が正しいこと |
| APIテスト | 全エンドポイント | リクエスト/レスポンスが仕様どおりであること |
| 画面遷移テスト | 全画面 | 画面遷移が定義どおりであること |

### 1.3 テスト結果の記録

| 列 | 内容 |
|----|------|
| 結果 | OK / NG / 未実施 |
| 実施日 | テスト実施日（YYYY/MM/DD） |
| 実施者 | テスト実施者名 |
| 備考 | NG時の事象・チケット番号等 |
